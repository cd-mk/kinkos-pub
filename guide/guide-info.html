영문 소문자, 숫자, 언더스코어 조합

  • 영문 소문자, 숫자, 언더스코어(_)만 사용할 수 있다.
  • 언더스코어(_)는 2개 이상의 단어를 조합할 때만 사용한다.
  • 단어와 숫자를 조합하는 경우 언더스코어(_)를 생략한다.
  • 언더스코어(_)를 이용하여 3단계를 초과하여 조합할 수 없다.

이미지 네이밍

  • 이미지 예약어는 네이밍 규칙에 근거하여 작성한다.
  • 같은 분류의 이미지가 두 개 이상이면 파일 이름 마지막에 숫자를 추가하여 구분한다.
  • 이미지 네이밍을 확장해야 할 때는 이미지 네이밍 조합 규칙을 준수한다.
  • 의미를 잘 살린 이미지의 naming은 이미지를 보지 않아도 대강의 의미를 알 수 있다.
  • 이는 규모가 큰 프로젝트일 경우 다수의 작업자들이 중복된 이미지를 저장하는 것에 대한 예방을 목적으로 한다.

HTML

  • 전체적인 마크업은 HTML5 을 기본으로 한다.
  • Tag는 각각의 의미에 맞는 마크업을 하여 StyleSheet가 적용되지 않더라도 내용이 제대로 전달될 수 있게 작성한다.
  • 모든 태그는 반드시 닫아준다. br, img, input 처럼 닫기 태그가 없는 것은 태그 끝에서 닫아준다.
  • 각 태그 속성 값은 항상 " " (따옴표)로 묶는다.
  • inline 스타일(< div style="....." > )은 되도록이면 자제한다.
  • img Tag는 해당 요소를 대체하기 위해 alt 속성을 기본적으로 사용한다. (border는 css에서 정의한다.)
  • font 태그 대신 span 태그를 사용한다.
  • 강조표현은 strong 태그를 사용한다.
